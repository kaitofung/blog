Java对象内存布局

发表于 2017-08-26   |  

一个Java对象的内存布局包括：对象头，对象的实例数据，还有对齐填充位。

对象头：对象头里面有MarkWord,ClassPointer,ArrayLength。

  • Mark Word用于存储对象自身的运行时数据，如哈希码（HashCode）、GC分代年龄、锁状态标志、线程持有的锁、偏向线程ID、偏向时间戳等等。在64位虚拟机位8字节，32位虚拟机位4字节。

  • 用来指向对象对应的Class对象（其对应的元数据对象）的内存地址。在32位系统占4字节，在64位系统中占8字节。

  • 如果对象是数组，则保持数组的长度。占4字节

对象实际数据包括了对象的所有成员变量，其大小由各个成员变量的大小决定，比如：byte和boolean是1个字节，short和char是2个字节，int和float是4个字节，long和double是8个字节，reference是4个字节（64位系统中是8个字节）。

对齐填充：对象的大小（ShadowSize）必须是8的整数倍的字节，不足则用此填充。

简单对象大小计算

通过classmexer.jar辅助得到对象大小。
运行设置JVM参数：

-javaagent:你自己的路径/classmexer.jar
-XX:+UseCompressedOops

public class Test{
    int a = 0;
    public static void main(String[] args) {
        System.out.println("Shallow Size: " + MemoryUtil.memoryUsageOf(new Test()) + " bytes");
        System.out.println("Retained Size: " + MemoryUtil.deepMemoryUsageOf(new Test()) + " bytes");
    }
}

在64位虚拟机中，Shallow Size=8+4+4+2=16，8为MarkWord的大小，第一个4为ClassPoiner的大小（64位虚拟机默认开启指针压缩，而32位虚拟机不支持压缩），第二个4为一个int数据类型的大小，因为不足16（8的整数倍）所以再加2.
